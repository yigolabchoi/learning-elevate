# Learning Elevate - 문서 센터

Learning Elevate 프로젝트의 모든 문서를 관리하는 중앙 저장소입니다.

## 📚 문서 구조

### 1. [기획 문서](./planning/)
서비스 기획, 기능 명세, 사용자 시나리오 등 제품 기획 관련 문서

- [서비스 기획서](./planning/service-plan.md) - 서비스 개요 및 핵심 가치
- [기능 명세서](./planning/feature-specification.md) - 상세 기능 정의
- [사용자 시나리오](./planning/user-scenarios.md) - 사용자별 시나리오

### 2. [디자인 시스템](./design/)
UI/UX 디자인 가이드라인 및 컴포넌트 라이브러리

- [디자인 시스템](./design/design-system.md) - 색상, 타이포그래피, 간격 등
- [컴포넌트 라이브러리](./design/component-library.md) - 재사용 가능한 UI 컴포넌트
- [화면 설계서](./design/screen-design.md) - 주요 화면 구조 및 IA

### 3. [개발 가이드](./development/)
개발 환경 설정, 코딩 컨벤션, 아키텍처 문서

- [개발 환경 설정](./development/setup-guide.md) - 로컬 개발 환경 구축
- [프로젝트 구조](./development/project-structure.md) - 폴더 구조 및 파일 구성
- [코딩 컨벤션](./development/coding-conventions.md) - 코드 작성 규칙
- [데이터베이스 스키마](./development/database-schema.md) - DB 설계 및 ERD

### 4. [API 문서](./api/)
백엔드 API 명세 및 통신 규격

- [API 명세서](./api/api-specification.md) - RESTful API 엔드포인트 정의

## 🚀 시작하기

1. **기획자/PM**: [서비스 기획서](./planning/service-plan.md)부터 시작하세요
2. **디자이너**: [디자인 시스템](./design/design-system.md)을 참고하세요
3. **개발자**: [개발 환경 설정](./development/setup-guide.md)으로 시작하세요

## 📝 문서 작성 원칙

- 모든 문서는 Markdown 형식으로 작성
- 변경 사항은 버전 히스토리에 기록
- 문서 간 상호 참조 시 상대 경로 링크 사용
- 이미지/다이어그램은 각 폴더의 `assets/` 하위에 저장

## 🔄 문서 업데이트 프로세스

1. 문서 수정 필요 시 해당 문서의 "변경 이력" 섹션에 날짜와 내용 기록
2. 주요 변경 사항은 팀에 공유
3. 정기적으로 문서 리뷰 및 업데이트

## 📞 문의

문서 관련 문의사항은 프로젝트 관리자에게 연락하세요.
