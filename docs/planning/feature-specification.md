## 1. 교사 커리큘럼 등록 기능

### 1-1. 기능 설명

교사가 자신의 수업 커리큘럼(단원, 차시, 학습 목표, 핵심 개념, 예시 문제 등)을 서비스에 입력하거나 업로드하면,
시스템이 이를 기반으로 문제 생성·레벨링에 활용합니다.

### 1-2. 주요 요소

#### 과목 (Subject)
- **MVP**: 영어(English)로 제한
- **향후**: 수학, 국어, 과학 등 확장

#### 학년/레벨 (Grade/Level)
- 초등 고학년 (Elementary High)
- 중학교 1-3학년 (Middle School 1-3)
- 고등학교 1-3학년 (High School 1-3)
- 토플/토익 준비 (TOEFL/TOEIC Prep)
- 커스텀 레벨 (Custom Level)

#### 단원/모듈 (Unit/Module)
영어 과목 예시:
- 기초 문법 (Basic Grammar)
- 시제 (Tenses)
- 전치사 (Prepositions)
- 독해 (Reading Comprehension)
- 어휘 (Vocabulary)
- 작문 (Writing)

#### 각 단원별 정보
- **학습 목표** (Learning Objectives): 문장 기반 설명
- **필수 개념/문법 포인트** (Key Concepts): 리스트 형태
- **예시 문장/예시 문제** (Example Sentences/Questions): 선택 사항

### 1-3. 교사 입력 방식

#### MVP (Phase 1)
- ✅ **양식 기반 입력** (웹 폼)
  - 단원명, 학습 목표, 핵심 개념 입력 필드
  - 예시 문제 추가 (선택)

#### 향후 버전 (Phase 2+)
- 📄 엑셀/문서 업로드
- 📋 기존 커리큘럼 템플릿 제공
  - 예: "중1 영어 기초 문법 템플릿"

## 2. 단계별 문제 자동 생성 기능

### 2-1. 기능 설명

시스템이 커리큘럼 정보를 바탕으로, 각 단원/레벨별로 다양한 유형의 문제를 자동 생성합니다.
같은 교육 목적을 가진 비슷한 문제를 여러 버전으로 생성해, 반복 훈련이 가능하도록 합니다.

### 2-2. 문제 유형

#### MVP 지원 유형
- ✅ **객관식** (Multiple Choice)
  - 4지선다 기본
  - 정답 1개
- ✅ **단답형** (Short Answer)
  - 단어/짧은 구문 입력
  - 정답 여러 개 허용 (동의어 인정)

#### 향후 지원 유형
- 📝 서술형 (Essay)
- 🎤 음성 답변 (Speaking)
- 👂 듣기 문제 (Listening)
- 🔗 매칭 문제 (Matching)

### 2-3. 수준 조절

#### 레벨 시스템
- **초급** (Beginner): Level 1-3
- **중급** (Intermediate): Level 4-7
- **고급** (Advanced): Level 8-10

#### 레벨별 특징
| 레벨 | 문장 복잡도 | 어휘 수준 | 문법 범위 |
|------|------------|----------|----------|
| 1-3 | 단문 중심 | 기초 어휘 | 기본 시제 |
| 4-7 | 복문 포함 | 중급 어휘 | 복합 문법 |
| 8-10 | 복잡한 구조 | 고급 어휘 | 고급 문법 |

### 2-4. 문제 수량 설정

- **기본값**: 단원별 10문항
- **확장**: 최대 20문항
- **교사 조정 가능**: 5~30문항 범위

### 2-5. 유사 문제 생성

같은 개념을 다른 예시로 반복 출제:

**예시: 현재완료 시제**
- 문제 1: "I ___ (live) here for 5 years."
- 문제 2: "She ___ (study) English since 2020."
- 문제 3: "They ___ (work) together for a long time."

### 2-7. AI 프롬프트 구조 (예시)

```
당신은 영어 교육 전문가입니다.
다음 정보를 바탕으로 문제를 생성해주세요:

- 단원: {unit_name}
- 학습 목표: {learning_objectives}
- 핵심 개념: {key_concepts}
- 난이도: {difficulty_level}
- 문제 유형: {question_type}
- 문제 수: {question_count}

3. 오답 선택지는 흔한 실수를 반영해야 합니다
4. 문제 간 중복을 피하되, 같은 개념을 다양한 맥락에서 출제하세요
```

## 3. 학생 레벨 기반 문제 풀이 기능

### 3-1. 기능 설명

학생은 본인의 계정으로 로그인 후,
현재 배정된 단원/레벨의 문제 세트를 풀며,
서비스는 학생의 과거 성과를 참고해 난이도를 자동 조정합니다.

### 3-2. 로그인/인증

#### 계정 생성 방식
- **교사 초대**: 교사가 학생 이메일/ID로 초대
- **초대 링크**: 교사가 생성한 링크로 가입
- **학부모 연동**: 학부모 계정과 자동 연결

#### 학생 프로필
```typescript
interface StudentProfile {
  id: string;
  name: string;
  email: string;
  gradeLevel: string;
  currentLevel: number; // 1-10
  teacherId: string;
  parentId?: string;
  enrolledDate: Date;
}
```

### 3-3. 학습 경로

#### 1) 오늘의 학습 (Today's Learning)
- 교사가 지정한 과제
- 마감일 표시
- 우선순위 순서

#### 2) 추천 학습 (Recommended)
- AI가 약점 기반 추천
- 복습이 필요한 단원
- 다음 단계 학습

### 3-4. 문제 풀이 UI

```
[문제 풀이 화면]
┌─────────────────────────────────────┐
│ 현재완료 시제 연습                   │

│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
├─────────────────────────────────────┤
│                                     │
│ Q3. 다음 문장의 빈칸에 알맞은 형태를 │
│     쓰세요.                         │
│                                     │
│ I ___ (live) in Seoul for 5 years. │
│                                     │
│ 답: [________________]              │
│                                     │
│ [이전] [건너뛰기] [제출]            │
│                                     │
│ 💡 힌트 보기                        │
└─────────────────────────────────────┘
```

### 3-5. 풀이 시간 측정

- **문제별 소요 시간** 기록
- **전체 세트 소요 시간** 기록
- **시간 제한** (선택적, 교사 설정)

### 3-6. 적응형 학습 (Adaptive Learning)

```
학생 성과 → AI 분석 → 난이도 조정

예시:
- 최근 3개 세트 정답률 90% 이상 → 레벨 +1
- 최근 3개 세트 정답률 60% 이하 → 레벨 -1
- 특정 개념 반복 오답 → 해당 개념 집중 문제 추가
```

## 4. AI 답안 분석 및 설명 기능

### 4-1. 기능 설명

학생이 문제를 제출하면, AI가 즉시 답안을 채점하거나 정오 판단을 내리고,
틀린 문항/불완전한 답변에 대해 상세한 설명과 예시를 제공합니다.

### 4-2. 채점 방식

#### 객관식 (Multiple Choice)
- ✅ 정답/오답 자동 판별
- 즉시 결과 표시

#### 단답형 (Short Answer)
- 정확한 정답 매칭
- 동의어/유사 표현 인정
- 대소문자 구분 옵션

**예시**:
```
정답: "have lived"
인정되는 답:
- "have lived"
- "have been living" (현재완료 진행형도 인정)
- "'ve lived" (축약형 인정)
```

#### 서술형 (Essay) - 향후 버전
- 의미 유사도 기반 채점
- 문법/철자 오류 감점
- 내용 완성도 평가

### 4-3. 피드백 내용

#### 1) 왜 틀렸는지 (Why Wrong)
```
❌ 오답: "lived"

설명:
현재완료 시제는 "have/has + 과거분사" 형태로 사용됩니다.
"for 5 years"는 기간을 나타내므로 현재완료를 사용해야 합니다.
단순 과거 "lived"는 과거의 완료된 동작을 나타내므로 적절하지 않습니다.
```

#### 2) 정답 예시 (Correct Answer)
```
✅ 정답: "have lived"

예문:
- I have lived in Seoul for 5 years.
  (나는 서울에 5년 동안 살아왔다.)
- She has worked here since 2020.
  (그녀는 2020년부터 여기서 일해왔다.)
```

#### 3) 추가 연습 문제 (Optional)
```
💪 비슷한 문제를 더 풀어볼까요?

1. They ___ (study) English for 3 months.
2. He ___ (be) a teacher since 2015.

[지금 풀기] [나중에]
```

필요 시 수정하거나 보완 코멘트를 추가한 뒤,
최종 확정된 피드백이 학생/학부모에게 노출됩니다.

## 6. 학습 정보 저장 및 약점 추적 기능

### 6-1. 기능 설명

학생의 모든 학습 이력(문항 단위 성적, 소요 시간, 틀린 패턴 등)을 저장하고,
이를 기반으로 반복적으로 부족한 영역을 추천·보완합니다.

### 6-2. 저장 데이터

#### 학생 프로필
```typescript
interface StudentProfile {
  id: string;
  name: string;
  gradeLevel: string;
  currentLevel: number;
  teacherId: string;
  parentId?: string;
}
```

#### 시도 기록
```typescript
interface AttemptRecord {
  id: string;
  studentId: string;
  questionSetId: string;
  attemptDate: Date;
  score: number;
  totalQuestions: number;
  accuracyRate: number;
  timeSpent: number; // seconds
  answers: AnswerDetail[];
}

interface AnswerDetail {
  questionId: string;
  studentAnswer: string;
  correctAnswer: string;
  isCorrect: boolean;
  timeSpent: number;
  conceptTags: string[];
}
```

#### 개념 태그
```typescript
interface ConceptTag {
  id: string;
  name: string;
  category: string;
  subject: string;
  relatedTags: string[];
}

// 예시: 영어 문법 태그
const grammarTags = [
  { id: 'present-perfect', name: '현재완료', category: 'tense' },
  { id: 'preposition', name: '전치사', category: 'grammar' },
  { id: 'word-order', name: '어순', category: 'syntax' },
  // ...
];
```

### 6-3. 취약 영역 분석

#### 분석 알고리즘
```
1. 최근 N회 시도 데이터 수집 (N = 10)
2. 개념 태그별 정답률 계산
3. 정답률 60% 이하 태그를 "취약 태그"로 분류
4. 빈도와 최근성을 고려한 우선순위 산정
5. TOP 3~5 취약 개념 추출
```

#### 취약 태그 데이터 구조
```typescript
interface WeaknessSummary {
  studentId: string;
  weakConcepts: WeakConcept[];
  lastUpdated: Date;
}

interface WeakConcept {
  conceptTag: string;
  accuracyRate: number;
  attemptCount: number;
  lastAttemptDate: Date;
  priority: 'high' | 'medium' | 'low';
  recommendedActions: string[];
}
```

## 7. 학부모용 페이지/리포트 기능

### 7-1. 기능 설명

학부모는 자녀의 학습 상태를 요약·시각화된 형태로 확인할 수 있습니다.
세부 문항 피드백보다는, 전반적인 수준과 변화 추이를 중심으로 제공합니다.
